# Cube Code - 魔方码

> [English](./README.md) | [中文](./README.zh-CN.md)

一个 3D 二维码系统，将数据分布在立方体的六个面上，提供约 6 倍于标准二维码的数据容量；也支持独立普通二维码模式，方便用任意手机扫码器直接读取。

|------|------|------|

| Android | ✅ 已支持 | [Releases](https://github.com/MonkiFantasy/cube-code/releases) |

## 功能

### 核心功能

- **魔方码编码/解码** — 6 面数据分割，支持 L/M/Q/H 纠错级别；URL 输入会被标记并解码为可点击链接
- **十字展开图** — 标准立方体展开布局，清晰展示 6 个面
- **逐面查看** — 左右切换查看每个面的 QR 细节
- **多面重组** — 普通模式固定 6 面，按面 ID 排序后重组完整数据
- **独立模式** — 每个使用的面都是普通二维码，可被系统相机/微信等普通扫码器直接读取
- **独立模式面数** — 仅在独立模式下选择 1–6 个二维码面
- **空面图片** — 独立模式下未使用的面可上传自定义图片填充

### 3D 渲染

- **交互式 3D 立方体** — 拖拽旋转，Three.js 渲染
- **面导航按钮** — 前/后/上/下/左/右快速切换视角
- **多种颜色模式** — 炫彩/黑白/异色/异色炫彩
- **QR 中心图标** — 可自定义二维码中心图标
- **玻璃材质** — 透明亚克力效果
- **基因码材质** — 紫/红/蓝三色像素浮雕立方体，使用 QR 真实内容边界贴合立方体面

### 移动端 & Android

- **响应式 UI** — 手机端优化布局
- **PWA 可安装** — 添加到主屏幕，类原生体验
- **Android APK** — GitHub Actions CI/CD 自动构建
- **原生保存图片** — 长按保存 QR 图片到相册
- **摄像头扫描** — 实时魔方码扫描，也支持普通二维码扫描
- **上传图片解码** — 从相册选择魔方码十字图或普通二维码图片解码

### 国际化

- **中英文** — 完整 i18n 支持

## 概念

传统二维码是 2D 的 — 数据编码在单一平面上。魔方码将数据分布在立方体的六个面上：

- 普通模式下，每个面承载一个带魔方码协议的二维码，包含完整数据的一部分
- **3-bit 标志位**（001–110）标识每个面的位置
- 扫描全部六个面后，按面 ID 排序拼接还原完整数据
- 独立模式下，每个使用的面直接编码原始文本，是普通二维码，不使用魔方码协议封装

```
        [3]
  [5]  [1]  [6]  [2]
        [4]
```

## 数据容量

| 模式       | 单个二维码 (V40) | 魔方码 (6 面) |
|------------|-----------------|---------------|
| 数字模式   | 7,089 字符      | ~42,534 字符  |
| 字母模式   | 4,296 字符      | ~25,776 字符  |
| 字节模式   | 2,953 字节      | ~17,718 字节  |

## 协议

普通魔方码模式下，每个面的二维码载荷：

```
[3-bit 面ID][13-bit 分片长度][数据分片]
```

扫描全部 6 个面并按面 ID 排序后，完整数据结构为：

```
[版本号][数据类型][内容][CRC16校验]
```

| 字段     | 大小    | 说明                              |
|----------|---------|-----------------------------------|
| 面 ID    | 3 bit   | 001–110，标识面的位置              |
| 分片长度 | 13 bit  | 当前面的数据分片长度               |
| 版本号   | 1 字节  | 协议版本                          |
| 数据类型 | 1 字节  | 0x00=文本, 0x01=二进制, 0x02=URL  |
| 内容     | N 字节  | 实际数据，拆分到 6 个面            |
| CRC16    | 2 字节  | 完整数据的校验和                   |

> 独立模式不使用上述协议封装；每个使用面直接生成普通 QR，内容就是原始输入文本。

## 开发

```bash
# 安装依赖
npm install

# 启动开发服务器（支持 HTTPS，可调用摄像头）
npm run dev

# 运行测试
npm test

# 代码检查
npm run lint

# 构建
npm run build
```

浏览器打开 https://localhost:5173（自签名证书，需手动信任）。

摄像头访问需要 HTTPS — 开发服务器自动使用自签名证书。

### Android 开发

```bash
npm run cap:sync         # 同步 web 资源到 Android
npm run cap:open:android # 在 Android Studio 中打开
```

### 发布

通过 GitHub Actions 自动化发布：

```bash
# 通过 tag
git tag v1.0.0
git push origin v1.0.0

# 或通过 GitHub Actions → Release Android APK → Run workflow
```
